> Este repositorio contiene la **documentación de criterios de aceptación** bajo el formato **Gherkin** para el proyecto **EntreTodos**. 

---

## 📋 Sobre el Proyecto
El objetivo es estandarizar los requisitos funcionales y no funcionales mediante **Behavior-Driven Development (BDD)**, garantizando que el equipo de desarrollo, QA y los interesados hablen el mismo lenguaje.

## 📂 Estructura de Escenarios
Los archivos están organizados por categorías según su naturaleza técnica:

| Archivo | Descripción |
| :--- | :--- |
| `tiempo_respuesta.feature` | HUNF-01: Rendimiento, cálculos y balances |
| `facilidad_uso.feature` | HUNF-02: UX y accesibilidad para usuarios |
| `compatibilidad_android.feature` | HUNF-03: Estándares de calidad en Android |
| `registro_rapido.feature` | HUNF-04: Eficiencia en el flujo de registro |

## 💡 Cómo visualizar los archivos
Para obtener la mejor experiencia de lectura en **VS Code**, te recomiendo instalar la siguiente extensión:

1. Ve a la pestaña de extensiones (`Ctrl+Shift+X`).
2. Busca e instala: **[Cucumber (Gherkin)](https://marketplace.visualstudio.com/items?itemName=alexkrechik.cucumberautocomplete)**.
3. Esto habilitará el resaltado de sintaxis y autocompletado en todos los archivos `.feature`.

---

## 🛠️ Convenciones de Escritura
Cada escenario sigue la estructura estándar **Given-When-Then** (Dado-Cuando-Entonces):

*   **Given (Dado):** El contexto o estado inicial.
*   **When (Cuando):** La acción específica realizada por el usuario.
*   **Then (Entonces):** El resultado esperado o validación.
